# Матрица проверок маршрутов и UX

## 1. Автоматический прогон API

Файл сценариев: `tools/route_scenarios.json` (30 пар точек, профили cyclist/pedestrian, ожидания `ok` / `point_outside` / `any`).

Запуск (API должен быть уже поднят, граф загружен):

```bash
cd NIR
python bike_router/tools/run_route_scenarios.py --base http://127.0.0.1:8000
```

Скрипт печатает TSV и завершается с кодом 1, если фактический HTTP-код и `detail.code` не совпали с полем `expect`. После смены `AREA_*` пересмотрите сценарии с краевыми координатами (s18, s29, s30).

Интерпретация колонок:

| Колонка | Смысл |
|--------|--------|
| `stairs_len_m`, `stairs_seg` | по первому варианту маршрута — есть ли лестницы |
| `na_surface_pct` | доля сегментов без `surface` в OSM (качество данных) |

## 2. Ручная проверка UX (чеклист)

Отметьте после проверки в браузере и на телефоне.

| Состояние | Ожидание |
|-----------|----------|
| Пустой экран / нет точек | Полоса статуса объясняет, что указать старт и финиш |
| Только старт | Статус просит финиш; кнопка построения неактивна |
| Две точки, маршрут не строили | Статус «Нажмите Построить» |
| Загрузка | Спиннер, статус «Строим маршрут», подсказка при долгом ответе |
| Маршрут не найден (NO_PATH) | Понятное сообщение в тосте |
| Точка вне зоны | Сообщение про зону / AREA_* |
| Геокодинг: пустой результат | Предупреждение «ничего не найдено» |
| Геокодинг: сеть / 502 | Отдельный текст про сеть и сервер |
| API выключен | Сообщение про недоступность сервера |
| Успех | Варианты, легенда, «Скопировать ссылку» |

## 3. Ссылка на маршрут

1. Построить маршрут, переключить вариант, сменить профиль.
2. Скопировать ссылку, открыть в новой вкладке / перезагрузить страницу.
3. Убедиться: те же точки на карте, тот же профиль, выбранный вариант, маршрут пересчитался.

## 4. Мобильная версия

Проверить на реальном устройстве: ввод адреса, кнопки ≥ 44px, карта + сайдбар не перекрывают друг друга, скролл панели, тосты читаемы.
