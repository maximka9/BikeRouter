О системе

Bike Router — веб-сервис построения и сравнения вариантов маршрутов для велосипедистов и пешеходов с учётом рельефа (SRTM), тегов дороги и покрытия в OpenStreetMap, а также оценки озеленения вдоль рёбер графа. Результаты носят информационный характер и не заменяют соблюдение ПДД и оценку безопасности в реальных условиях.

«Сборка графа» — момент последнего запуска загрузки OSM и пересчёта весов на этом сервере. Это не дата релиза данных на openstreetmap.org: фактическое состояние карты соответствует тому снимку, который вернул Overpass API в момент загрузки (с учётом кэша OSMnx).

Источники данных

  • OpenStreetMap — геометрия дорог, теги highway, surface, steps и др. Загрузка через OSMnx по полигону области.
  • Рельеф — цифровая модель высот для оценки уклонов вдоль рёбер (сервис в проекте; при отсутствии данных возможны упрощения).
  • Озеленение — при включённом анализе: спутниковые тайлы (конфигурация TMS) и оценка доли деревьев/травы в коридоре ребра. При недоступности PIL/тайлов используются заглушки.
  • Геокодирование — Nominatim (публичный сервис OSM Foundation) с ограничениями по частоте запросов; в интерфейсе поиск только по явному действию пользователя.

Как считаются веса

Для каждого ребра графа задаётся длина и перепад высоты. Базовый «физический» вес имеет вид base = mg × длина × (ν + эффективный уклон) с ограничением крутизны и отдельной обработкой длинных рёбер (интерполяция высот по сегментам ~30 м).

Затем вес умножается на коэффициенты по типу дороги (highway) и покрытию (surface) из справочника профиля. Если тега нет или значение неизвестно таблице, используется коэффициент 1.0 (нейтральная оценка).

Режим «с учётом озеленения» дополнительно умножает вес на функцию от спутниковой оценки зелёности ребра (чем выше зелёность, тем ниже вес в этом режиме).

Кратчайший вариант на карте строится по минимальной сумме длин рёбер OSM (length), а не по энергетическому весу.

Ограничения качества данных

  • OSM заполняется сообществом: в одних районах подробные surface и highway, в других — нет. Это влияет на достоверность «комфорта» и времени.
  • Отсутствие surface не означает плохую дорогу — в расчёте используется нейтральный коэффициент, но интерпретация для пользователя неоднозначна.
  • Рельеф и спутниковый анализ — модели с погрешностью; на коротких рёбрах возможны упрощения.
  • Маршрут оптимален относительно этой модели и этого графа, а не «истинной» дорожной ситуации в реальном времени.

Сервис показывает предупреждения, если по длине маршрута (сумма длин рёбер OSM) велика доля участков без surface, с коэффициентом покрытия или highway по умолчанию, а также при большом числе или суммарной длине лестниц.

Базовая карта и клиент

Подложка карты — растровые тайлы CARTO Voyager (сеть доставки CARTO, cartocdn.com). Атрибуция: © CARTO и © OpenStreetMap (данные OSM распространяются на условиях ODbL).

Отрисовка карты и взаимодействие — MapLibre GL JS (лицензия BSD-3-Clause); в этом проекте библиотека отдаётся с того же origin, что и страница (/static/vendor/maplibre/), без внешнего CDN на странице карты.

Публичный сервер tile.openstreetmap.org рассчитан на ограниченную нагрузку; для высоконагруженного продакшена рекомендуется собственный тайловый кэш, коммерческий провайдер или иная схема с явным соглашением об использовании — без массовой предзагрузки тайлов с инфраструктуры OSMF.

Атрибуция и условия использования внешних сервисов

  • OpenStreetMap — данные © участники OSM, ODbL; при отображении карты необходима заметная атрибуция OSM.
  • Nominatim (геокодирование) — сервис OSM Foundation; действуют политика использования (в т.ч. ограничение частоты запросов). В интерфейсе поиск выполняется только по явному действию пользователя (кнопка или Enter), без автодополнения на каждый символ.
  • CARTO — тайлы используются в соответствии с типовой атрибуцией поставщика; при смене подложки проверьте условия нового провайдера.

Проект не гарантирует актуальность данных OSM, точность рельефа и спутниковых оценок, бесперебойность публичных API и пригодность маршрутов для конкретного пользователя. Ответственность за выбор дороги и безопасность лежит на пользователе.
